# Iteration 0: Vibe Coding — Claims Form

## What will be built?

A standalone claims reporting form built with React and Vite, generated from a single one-shot prompt.

The form covers the same domain as the real implementation that follows — but without architecture, without tests, without integration into the existing system.

## Agentic Engineering Aspects

None — that is the point.

- no plan
- no requirements
- no architecture rules
- no tests
- no review
- no integration into the existing application

## Example Prompt

```
Build me a claims reporting form as a standalone web app with React and Vite.

Make it look professional with a modern UI. Add form validation and a success message after submission.
```

## Flow

1. Paste the prompt into Claude Code.
2. Let Claude generate the complete project in one shot.
3. Run it, show the result in the browser.
4. Discuss: what is missing for professional software?

## Focus

Vibe coding produces a working UI quickly. But it is not connected to the real policy data, has no validation against business rules, no persistence, no security, no tests, and no way to evolve safely inside an existing system.

## Learning

Vibe coding is fast output. It is not engineering. The rest of the workshop shows what engineering looks like when AI does the implementation inside a real application.
